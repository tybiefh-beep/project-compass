Claude Code 参考

CLAUDE.md 文件体系

Claude Code 启动时按以下优先级加载指令文件：

  • ~/.claude/CLAUDE.md — 全局个人指令，所有项目生效
  • ./CLAUDE.md — 项目根目录指令，随 Git 提交
  • ./.claude/CLAUDE.md — 项目隐藏指令（建议不提交）
  • ./.claude/rules/*.md — 规则目录，按文件名排序加载
  • CLAUDE.local.md — 个人偏好（应加入 .gitignore）

CLAUDE.md 内容结构

推荐用 Markdown 写，AI 直接读取。常见章节：

  • Project — 项目名称和一句话定义
  • User Profile — 用户技术水平、沟通偏好
  • Tech Stack — 语言、框架、运行环境
  • Architecture — 目录结构、模块划分
  • Commands — 构建/测试/运行命令
  • Code Style — 命名规范、注释风格
  • 自动模式 — 自主权限边界、确认点
  • Run Context — 模型架构说明（壳+脑）

@ 导入语法

用 @文件路径 导入其他文件内容，拼接到指令中：

# Project: 我的工具
@README.md
@docs/architecture.md

适合引用已有文档，避免维护两份相同内容。

.claude/rules/ 规则目录

当 CLAUDE.md 超过 200 行，应拆分到规则目录：

.claude/
├── CLAUDE.md          # 主文件，放核心指令
└── rules/
    ├── api-design.md  # API 设计规范
    ├── testing.md     # 测试规范
    └── git.md         # Git 工作流

规则文件自动被读取，可按文件名前缀控制排序。

YAML Frontmatter

CLAUDE.md 顶部可写 YAML frontmatter，用 paths 限定作用域：

---
paths: "src/api/**"
---
# API 模块规范
仅在处理 src/api/ 下文件时加载此指令。

paths 支持 glob 模式，让规则精准匹配文件范围。

SKILL.md 技能定义

放在项目根目录或 .claude/skills/ 下，XML 格式：

<skill>
  <name>code-review</name>
  <description>代码审查助手</description>
  <trigger>审查|review|code review</trigger>
  <anti-trigger>快速修复|hotfix</anti-trigger>
  <steps>
    1. 读取变更文件
    2. 检查安全隐患
    3. 审查代码风格
    4. 输出改进建议
  </steps>
</skill>

Skill 字段说明

  • <name> — 唯一标识，字母+连字符
  • <description> — 一句话描述，AI 用来理解用途
  • <trigger> — 触发关键词或正则，支持 | 分隔
  • <anti-trigger> — 排除场景，防止误触发
  • <steps> — 执行步骤，每步一行，顺序执行

Skill 触发机制

Claude Code 在每条用户消息前检查所有 skill：

  • 用户输入匹配 trigger → 加载该 skill 的 steps
  • 同时匹配多个 skill → 全部加载，按文件名排序
  • 匹配 anti-trigger → 排除该 skill，即使 trigger 命中
  • trigger 太宽 → 干扰正常对话；太窄 → 永不触发

关键：trigger 应足够具体，避免与系统内置行为重叠。

文件发现顺序

Claude Code 按此顺序加载文件（高优先级先）：

  1. ~/.claude/CLAUDE.md — 全局用户指令
  2. ./CLAUDE.md — 项目根目录指令
  3. ./.claude/CLAUDE.md — 项目隐藏指令
  4. ./.claude/rules/*.md — 规则文件
  5. ./SKILL.md — 技能定义
  6. ./.claude/skills/*.skill.md — 独立技能文件
  7. CLAUDE.local.md — 本地个人偏好

内存系统 (Memory)

持久化记忆，存储在 .claude/memory/：

  • user — 用户角色、偏好、知识背景
  • feedback — 用户反馈的工作方式
  • project — 项目进度、目标、决策记录
  • reference — 外部资源指针（文档链接等）

AI 在对话中自动读写记忆，跨 session 持久化。

最佳实践

  • CLAUDE.md 控制在 200 行内，超出拆分到 rules/
  • 每个 skill 的 trigger 写 3-5 个核心关键词
  • anti-trigger 宁可多写，减少误触发
  • steps 每步一个动作，不合并多步骤
  • CLAUDE.local.md 放个人偏好，加入 .gitignore
  • API Key、密码等敏感信息绝不写入 CLAUDE.md
